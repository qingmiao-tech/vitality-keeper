# 架构说明

元气守恒·筑基令采用 Tauri v2 + Rust + 原生 HTML/CSS/JavaScript。它的目标不是做一个复杂的任务管理系统，而是把“休息提醒、视频引导、多屏展示、系统托盘”这几个核心动作做轻、做稳。

## 核心模块

- Rust 后端：负责倒计时、托盘菜单、多窗口创建、开机自启、配置持久化和系统空闲检测。
- 前端控制中心：负责设置展示、视频源选择、语言切换、主题切换和播放列表预览。
- 休息页：负责本地视频播放、倒计时、进度线、暂停/静音/全屏同步。
- 配置层：通过 `tauri-plugin-store` 统一读写配置，避免前端 localStorage 与后端状态分叉。

## 休息流程

1. Rust 定时器持续追踪专注间隔。
2. 到达触发时间后，后端创建一轮 break flow，并选定本轮视频。
3. 后端为每个显示器创建一个无边框休息窗口。
4. 主休息窗口发布播放进度、暂停、静音和全屏状态。
5. 其他休息窗口订阅状态并保持同步。
6. 休息完成、跳过或推迟后，后端统一关闭窗口并重置状态。

## 设计取舍

- 只保留长休主流程，不保留短休分支，降低认知成本。
- 本地视频优先，不依赖云服务，不上传用户素材。
- 用 Tauri 自定义资源协议读取本地媒体，减少本地文件播放的跨域和自动静音问题。
- 当前 Windows 支持最完整；macOS 和 Linux 作为源码构建目标继续验证。
